Design Document — Window Washer Challenge (rev. B42-WW-0.7)
1) Концепция

Название: Window Washer
Слоган: “No hope of survival. Hanging by a thread.”
Тип: Solo Challenge (дух — «A Really CDDA»)
Питч: Старт на подвесной платформе мойщика окон на фасаде небоскрёба в Луисвилле. Внутрь — только через окна. Земля недостижима, внутри — толпы, наверху — ветер и одиночество. Блэкаут неизбежен; без питания — ручная лебёдка и борьба за каждый этаж.

Цель: прожить как можно дольше, планируя перемещение платформы, вылазки внутрь и доступ к энергии.

2) Стартовые условия

Мир/карта: Louisville, KY (world = "Muldraugh, KY").

Спавн: x=12785, y=1539, z=24 — средний сегмент фасада.

Время: утро, hourOfDay = 7.

Доступ вниз: из старта «не достаёт» до нижнего сегмента; требуется перестановка аутригеров сверху.

3) Профессия: Window Washer

Иконка: prof_WindowWasher

Образ: выносливый высотник; привык работать на открытых высотах.

Выдача очков: при выборе профессии игрок получает +8 очков для самостоятельного выбора перков/дебаффов.

Обязательные трейты: нет.

Единственная встроенная механика: HighLover (см. §5.4).

3.1 Стартовый лоадаут

Одежда (корректно по слотам):
Glasses_Sun, Socks_Ankle, Shoes_WorkBoots, Trousers_Denim, Tshirt_DefaultDECAL, Shirt_Lumberjack, Gloves_LeatherGloves, Hat_HardHat, Belt, Bag_FannyPackFront, WristWatch_Left_DigitalBlack.

Инвентарь:

4) Платформа (геометрия и безопасность)

Настил: 1×3 тайла (odd size = 3), ориентация по умолчанию EW, пол: constructedobjects_01_86.

Перила: вдоль длинной стороны + торцевые заглушки (construction_01_32/_38); наружная сторона настраивается.

Пульт: две центральные клетки настила. Любое движение только если игрок стоит на пульте.

5) Механика движения
5.1 Вертикальный шаг

Дискретно ±1 этаж, Timed Action.

Пайплайн:

Валидация всех целевых квадратов настила/перил.

Построение новой полосы пола/перил на целевом Z.

Перенос содержимого «ручья» каждой клетки:

World-предметы (с сохранением оффсетов).

Трупы IsoDeadBody с явным setSquare и добавлением в списки клетки (fallback при расхождениях).

Живые (зомби/NPC, кроме игрока) — телепорт + стабилизация Lx/Ly/Lz.

Простой декор-тайлы (кроме структур/окон/дверей/динамики).

Кровь: snapshot движковой крови → раскладка overlay_blood_floor (кластер вдоль платформы) → перенос уже существующих оверлеев → отложенная очистка движковой крови на старых клетках.

Снос старого пола/перил по сохранённым спискам.

Игрок: сохраняем X/Y, меняем только Z (+ стабилизация Lz).

5.2 Горизонталь — только через верх

На этажах нельзя сместиться вбок.

Процедура смены колонны: подняться на верх сегмента → контекст-действие «Переставить аутригеры» (занимает время; в будущем — инструменты/расходники) → продолжать вертикальную работу в новой колонне (в т.ч. спуск на верх нижнего сегмента).

5.3 Приводы

Электрическая лебёдка

Доступна при наличии питания (здание/генератор на крыше/техзоне).

Быстро, без затрат выносливости.

При блэкауте недоступна; снова доступна при покрытии генератором (реком. радиус 12–20 тайлов от узла аутригеров).

Хоткеи PgUp/PgDn вызывают электро-режим (если питание есть).

Ручная лебёдка

Всегда доступна.

Медленнее, тратит Endurance за этаж, с учётом перегруза.

При низкой выносливости — авто-отмена шага.

Параметры (тюнинг):

moveDurationElectric = 0.5 s/этаж

moveDurationManual = 1.5 s/этаж (вниз чуть быстрее)

Стоимость стамины (ручной):

База: manualPerFloor = 0.25 (25% полоски).

Множители: вверх ×1.00; вниз ×0.50.

Перки Strength/Fitness уменьшают цену (до −50%/−40%) и ускоряют действие (до −20%).

Перегруз: +0.0035 за каждый кг сверх лимита.

Низкая выносливость: <0.30 → +25% цены/времени; <0.15 → ×2 цена, ×1.5 длительность.

Порог авто-отмены: Endurance ≤ 0.12.

5.4 HighLover — «Любитель высоты»

Единственная встроенная «профильная» мета-механика профессии.

На высоте (z ≥ 7): мгновенно сбрасывает скуку — Boredom, BoredomLevel, BoredomChange = 0 на каждом тике проверки.

На земле: добавляет доп. прирост скуки (к BoredomChange), зависящий от кол-ва дней без высоты; кривая прироста ограничена локальным капом MaxExtraLevelPerMin = 0.015 (≈1.5% шкалы/мин).

Тик раз в ~3 сек; подробные логи при включённом дебаге.

6) Геймплей и ограничения

Вход — только через окна.

Срыв = смерть.

Старт — средний сегмент; вниз — лишь после перестановки аутригеров.

Внутренние лестницы не ломают челлендж: высокая популяция.

Жизнь на крыше/платформе с вылазками внутрь предположительно длительная.

Блэкаут усложняет навигацию; без генератора — только ручной режим.

Популяция/лут: таргет — эквивалент “A Really CDDA”.

Для отладки используются собственные SandboxVars; под релиз выровнять с профилем CDDA.

8) Управление и UI

Контекст-меню (на пульте): Move Scaffold → Up (+1) / Down (−1); отдельные пункты для Electric/Manual; debug-instant.

Хоткеи: PgUp / PgDn — только на пульте; по умолчанию запускают электро-привод (если питание доступно).

Подсказки (реком.):

«Электролебёдка доступна (питание/генератор)».

«Нет питания: ручной режим (тратит выносливость)».

«Чтобы работать ниже — переставьте аутригеры на верхнем уровне нижнего сегмента».

«Высота бодрит: скука снижается на верхних уровнях».

9) Атмосфера и аудио

Разные лупы для электро/ручной лебёдки.

Фоновый ветер/скрип высоты.

Кровь на платформе — кластерные оверлеи вдоль «ручья».

10) Техническая спецификация

Строительство/снос: buildPlatformAt / destroyPlatform + перила (линия/торцы).

Валидация: проверка всех клеток настила/перил (getSquare), иначе шаг отменяется.

Перенос: world-items (без item:setWorldItem(nil)), dead bodies (жёсткий rebind), live pawns (кроме игрока), декор (без структур/динамики), кровь (snapshot → overlays → deferred cleanup).

Игрок: X/Y неизменны; меняется Z (+ setLz).

Энергия: проверка питания/генератора (радиус покрытия настраиваемый; интеграция с генераторами — WIP).

События: контекст-меню, хоткеи, OnWeaponHitCharacter/OnZombieDead (кровь), OnPlayerDeath — останов движения.

Надёжность: критичные операции — в pcall; после тайл-операций — RecalcAllWithNeighbours(true).

12) QA чек-лист

Корректный спавн; платформа/перила построены верно.

Контекст-меню/горячие клавиши активны только на пульте.

Вертикальный шаг: X/Y сохраняются, меняется Z.

Перенос: world-items, трупы, живые (кроме игрока), декор — корректно.

Кровь: появляется кластером; после шага нет артефактов; движковая кровь очищается.

Старые перила/полы удаляются; новые — корректны.

Перестановка аутригеров переключает колонну.

Электро/ручной режимы соответствуют условиям питания; ручной — списывает стамину и учитывает перегруз; авто-отмена при низкой выносливости работает.

HighLover: на высоте скука сбрасывается; на земле — растёт по кривой с капом.

Производительность стабильна после серии шагов.

13) Roadmap (v2)

Инструменты/расходники для перестановки аутригеров; шанс поломки/ивенты ремонта.

Погодные модификаторы (ветер/ливень) → увеличение времени/стоимости.

Небольшие горизонтальные смещения с риском срыва.

Требования к генератору (мощность/топливо) для электролебёдки.

Мини-ивенты на крыше/карнизах (ключ-карты, редкие NPC, кэш-лут).

Доп. визуал: канаты/тросы как отдельные тайлы (N/W-варианты, 128×256 px @2×, PNG-32).

14) Лор / ролик

Флэшбеки первого дня, блэкаут, запуск генератора, бои на карнизе. В «сейчас» — рана/подозрение на заражение. Обрыв: Hanging by a thread.
